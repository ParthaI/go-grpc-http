Section 1: Fundamentals

Q1

What is a goroutine?

▼

A goroutine is a lightweight, user-space thread managed by the Go runtime. It runs concurrently with other goroutines in the same address space.

func main() {
    go sayHello() // spawns a goroutine — runs concurrently
    fmt.Println("main function")
    time.Sleep(time.Second)
}

func sayHello() {
    fmt.Println("hello from goroutine")
}
Go
main function
hello from goroutine

Key facts:

  • Starts with ~2KB stack (grows dynamically up to 1GB)
  • Costs ~few microseconds to create (vs milliseconds for OS threads)
  • You can run millions of goroutines on a single machine
  • Managed by Go's runtime scheduler, not the OS

Q3

What happens when the main goroutine exits?

▼

When main() returns, the entire program exits immediately — all other goroutines are killed, even if they haven't finished.

func main() {
    go func() {
        time.Sleep(2 * time.Second)
        fmt.Println("This will NEVER print")
    }()

    fmt.Println("main exits immediately")
}
Go
main exits immediately
Fix: Use sync.WaitGroup, channels, or select to wait for goroutines.

Q6

What are channels and how do goroutines communicate?

▼

Channels are typed conduits for sending and receiving values between goroutines. Go's philosophy: "Don't communicate by sharing memory; share memory by communicating."

func main() {
    ch := make(chan int)

    // Producer goroutine
    go func() {
        ch <- 42 // send value into channel
    }()

    // Consumer (main goroutine)
    value := <-ch // receive value from channel
    fmt.Println(value) // 42
}
Go
ch <- value   // Send — blocks until someone receives
value := <-ch // Receive — blocks until someone sends
close(ch)     // Close — signals no more values
Go

Q11

Explain the select statement

▼

select lets a goroutine wait on multiple channel operations simultaneously:

select {
case msg := <-ch1:
    fmt.Println("from ch1:", msg)
case msg := <-ch2:
    fmt.Println("from ch2:", msg)
case <-time.After(5 * time.Second):
    fmt.Println("timeout")
default:
    fmt.Println("no channel ready") // non-blocking
}
Go
  • Multiple cases ready → one chosen randomly (fair)
  • No case ready, no default → blocks
  • With default → non-blocking

Q14

sync.RWMutex — multiple readers OR one writer

▼ 
type Cache struct {
    mu    sync.RWMutex
    items map[string]string
}

func (c *Cache) Get(key string) (string, bool) {
    c.mu.RLock()         // shared read lock — multiple readers OK
    defer c.mu.RUnlock()
    val, ok := c.items[key]
    return val, ok
}

func (c *Cache) Set(key, value string) {
    c.mu.Lock()          // exclusive write lock — blocks all
    defer c.mu.Unlock()
    c.items[key] = value
}
Go
Rule: Use RWMutex when read-to-write ratio is >10:1.

Fan-Out / Fan-In Pattern

▼ 
func fanIn(channels ...<-chan int) <-chan int {
    var wg sync.WaitGroup
    merged := make(chan int)

    for _, ch := range channels {
        wg.Add(1)
        go func(c <-chan int) {
            defer wg.Done()
            for val := range c {
                merged <- val
            }
        }(ch)
    }

    go func() {
        wg.Wait()
        close(merged)
    }()
    return merged
}

Worker Pool Pattern

func workerPool(numWorkers int, jobs <-chan Job) <-chan Result {
    results := make(chan Result)
    var wg sync.WaitGroup

    for w := 0; w < numWorkers; w++ {

        go func(id int) {
            defer wg.Done()
            for job := range jobs {

The GMP Model (Goroutine, Machine, Processor)

▼ 
// G = Goroutine (unit of work)
// M = Machine  (OS thread)
// P = Processor (scheduling context, local run queue)
//
// P0 [G1, G2, G3] → M0 (thread)
// P1 [G4, G5]     → M1 (thread)
// Global Queue: [G6, G7]
//
// - Each P has local run queue (max 256 Gs)
// - Empty P steals from other Ps (work-stealing)
// - GOMAXPROCS = number of Ps (default = CPU cores)

fmt.Println(runtime.NumGoroutine()) // active goroutines
runtime.Gosched()                  // yield to scheduler

Q62

Real-world: Production-grade concurrent fan-out pattern

▼ 
func listResources(ctx context.Context, regions []string) ([]Resource, error) {
    g, ctx := errgroup.WithContext(ctx)
    g.SetLimit(5) // max 5 concurrent region queries

    var mu sync.Mutex
    var all []Resource

    for _, region := range regions {
        r := region // capture loop var
        g.Go(func() error {
            resources, err := fetchRegion(ctx, r)
            if err != nil {
                return fmt.Errorf("region %s: %w", r, err)
            }
            mu.Lock()
            all = append(all, resources...)
            mu.Unlock()
            return nil
        })
    }
    return all, g.Wait()
}
Go

What makes this production-quality:

  • ✔ Bounded concurrency (SetLimit(5))
  • ✔ Error propagation (first error cancels all)
  • ✔ Context cancellation (user can cancel SQL query)
  • ✔ Thread-safe result collection (mutex)
  • ✔ Error wrapping with region context

Q63

Race condition vs data race — what's the difference?

▼

Data race — two goroutines access the same variable concurrently, at least one writes, no synchronization. Detected by go run -race.

Race condition — program correctness depends on ordering of operations. NOT detected by the race detector.

// DATA RACE — detected by -race
var counter int
go func() { counter++ }()
go func() { counter++ }()

// RACE CONDITION (no data race!) — NOT detected
// TOCTOU: Time-of-check to time-of-use
func withdraw(amount int) {
    mu.Lock()
    ok := balance >= amount
    mu.Unlock()
    // <-- another goroutine can change balance here!
    if ok {
        mu.Lock()
        balance -= amount // might go negative!
        mu.Unlock()
    }
}

// FIX: hold lock for entire check-and-act
mu.Lock()
defer mu.Unlock()
if balance >= amount { balance -= amount }
Go
Key point: -race catches data races but NOT race conditions.

Tee channel — split one channel into two

func tee(ctx context.Context, in <-chan int) (<-chan int, <-chan int) {
    out1 := make(chan int)
    out2 := make(chan int)
    go func() {
        defer close(out1); defer close(out2)
        for val := range in {
            o1, o2 := out1, out2
            for i := 0; i < 2; i++ {
                select {
                case o1 <- val: o1 = nil
                case o2 <- val: o2 = nil
                case <-ctx.Done(): return
                }
            }
        }
    }()
    return out1, out2
}

Bridge channel — flatten channel of channels

▼ 
func bridge(ctx context.Context, chanCh <-chan <-chan int) <-chan int {
    out := make(chan int)
    go func() {
        defer close(out)
        for {
            var ch <-chan int
            select {
            case maybeCh, ok := <-chanCh:
                if !ok { return }
                ch = maybeCh
            case <-ctx.Done(): return
            }
            for val := range ch {
                select {
                case out <- val:
                case <-ctx.Done(): return
                }
            }
        }
    }()
    return out
}

Future/Promise pattern with generics

▼ 
type Future[T any] struct {
    ch     chan struct{}
    result T
    err    error
}

func NewFuture[T any](fn func() (T, error)) *Future[T] {
    f := &Future[T]{ch: make(chan struct{})}
    go func() {
        f.result, f.err = fn()
        close(f.ch) // signal completion
    }()
    return f
}

func (f *Future[T]) Get() (T, error) {
    <-f.ch
    return f.result, f.err
}

// Fire 3 async calls, block only when needed
user := NewFuture(func() (*User, error) { return fetchUser(id) })
orders := NewFuture(func() ([]Order, error) { return fetchOrders(id) })

Concurrent-safe LRU cache

▼ 
type LRUCache struct {
    mu       sync.Mutex
    capacity int
    items    map[string]*list.Element
    order    *list.List
}

func (c *LRUCache) Get(key string) (interface{}, bool) {
    c.mu.Lock()
    defer c.mu.Unlock()
    if elem, ok := c.items[key]; ok {
        c.order.MoveToFront(elem)
        return elem.Value.(*entry).value, true
    }
    return nil, false
}

func (c *LRUCache) Put(key string, value interface{}) {
    c.mu.Lock()
    defer c.mu.Unlock()
    if elem, ok := c.items[key]; ok {
        c.order.MoveToFront(elem)
        elem.Value.(*entry).value = value
        return
    }
    if c.order.Len() >= c.capacity {
        oldest := c.order.Back()
        c.order.Remove(oldest)
        delete(c.items, oldest.Value.(*entry).key)
    }
    elem := c.order.PushFront(&entry{key, value})
    c.items[key] = elem
}
